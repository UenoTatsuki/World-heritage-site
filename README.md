# 世界遺産マップ（World Heritage Map）

世界中のユネスコ世界遺産を、インタラクティブな地図とビジュアルなギャラリーから探せるWebアプリケーションです。学習およびポートフォリオ目的で、React + TypeScript を用いてゼロから開発しました。

🔗 **WebサイトURL**: https://world-heritage-site-pi.vercel.app/

 ![トップページ](./docs/screenshot-home.png)

---

## 概要

世界遺産に興味を持つ人が、地域や分類から遺産を探し、自分だけの「行きたいリスト」を作れるサービスです。地図からの探索と、写真主役のギャラリー閲覧という2つの導線を用意し、目的に応じて使い分けられる構成にしました。

---

## 主な機能

- **オープニング演出のあるトップページ**: 黒背景からキャッチコピーが浮かび上がり、複数の風景写真がクロスフェードで切り替わるヒーローを備えています。
- **インタラクティブ世界地図**: 各遺産の位置にピンを表示し、クリックで詳細を確認できます。地図は世界1個分に固定し、繰り返しスクロールを防いでいます。
- **多階層カテゴリー検索**: 地域・分類（文化／自然／複合）・登録年代を、チェックボックスで複数選択して絞り込めます。選択中の条件はタグで表示され、個別に解除できます。
- **行きたいリスト（要ログイン）**: 気になる遺産をお気に入り登録できます。お気に入り操作時にはログインを求める仕組みです。
- **ユーザー認証**: メールアドレスとパスワードによる新規登録・ログイン・ログアウトに対応しています。
- **画像の遅延読み込み**: 画面に表示された遺産の画像だけを順次読み込むことで、件数が増えても初期表示を軽く保ちます。
- **レスポンシブ対応**: スマートフォンでもレイアウトが崩れないよう、画面幅に応じて表示を最適化しています。

## データの出所

世界遺産のデータ（名称・座標・登録年・画像）は [Wikidata](https://www.wikidata.org/) から取得しています。Wikidataのデータはクリエイティブ・コモンズ CC0（パブリックドメイン）で提供されており、自由に利用・改変・再配布が可能です。画像は Wikimedia Commons のものを使用しています。

---

## こだわった点・工夫

- **コストを意識した技術選定**: 地図機能において、無料枠に制限のある商用APIではなく、完全無料の Leaflet + OpenStreetMap を採用しました。
- **データの権利への配慮**: 当初検討したデータ源は再配布に許可が必要だったため、ライセンスを調査し、CC0で自由に使える Wikidata を選定しました。
- **パフォーマンスの最適化**: Intersection Observer を用いた画像の遅延読み込みにより、表示する遺産が増えても初期表示が重くならない設計にしています。また画像URLにサイズ指定を付与し、読み込みを高速化しています。
- **影響範囲を限定した設計**: 一覧ページの複数選択フィルターは、既存の地図ページに影響を与えないよう独立した状態管理として実装しました。
- **再利用性を意識したコンポーネント設計**: 遺産カードやグリッドを、表示する場所に応じて柔軟に使い回せるよう設計しています。
- **国際化を意識したデータ設計**: 表示用の日本語名と、画像検索・データ照合用の英語名を分けて持つことで、表示と内部処理を分離しています。

## ローカルでの起動方法

### リポジトリをクローン
git clone https://github.com/your-name/world-heritage-site.git

cd world-heritage-site

### 依存パッケージをインストール
npm install

### 環境変数を設定（.envファイルを作成し、Firebaseの設定値を記入）
VITE_FIREBASE_API_KEY=...

VITE_FIREBASE_AUTH_DOMAIN=...

など

### 開発サーバーを起動
npm run dev

起動後、ブラウザで \`http://localhost:5173\` を開いてください。

※ 認証機能を利用するには、Firebaseプロジェクトの作成と \`.env\` への設定値の記入が必要です。
